.. _documentation_building:

***********************
Documentation Building
***********************

``scaldys-builder build windows docs`` drives Sphinx to produce two separate
HTML outputs from your project: a *user guide* and a *developer guide*.

Requirements
============

Install the ``[docs]`` extra::

    uv add --dev "scaldys-builder[docs]"

This installs Sphinx and the ``sphinx-rtd-theme``.

Expected project layout
========================

``scaldys-builder`` expects your Sphinx project to live under
``docs/manual/`` in the project root, with source files at
``docs/manual/source/``:

.. code-block:: text

    docs/
        manual/
            source/
                conf.py         ← Sphinx configuration
                index.rst       ← documentation root
                ...             ← additional RST pages
            Makefile            ← generated by sphinx-quickstart
            make.bat            ← generated by sphinx-quickstart

If you have not set up Sphinx yet, run ``sphinx-quickstart docs/manual`` to
generate the scaffolding.

User guide
==========

The user guide is built from the ``docs/manual/`` project in two formats:

**Multi-page HTML** (``dirhtml`` builder)::

    build/manual/html/

A standard multi-directory HTML site suitable for hosting on a web server or
browsing locally.

**Single-page HTML** (``singlehtml`` builder)::

    build/manual/singlehtml/

All pages merged into a single ``index.html`` file.  ``scaldys-builder``
copies this output into ``dist/pyinstaller/docs/`` so the user guide is
bundled inside the Windows installer and can be opened offline.

Developer guide
===============

The developer guide is generated automatically from your source code's
docstrings and then compiled to HTML.

**Step 1 — sphinx-apidoc**

``sphinx-apidoc`` scans the source tree (``source_root`` as configured in
``builder.toml``, default ``src/``) and generates ``.rst`` stub files in a
temporary staging directory.

**Step 2 — Sphinx HTML build**

The generated stubs are compiled to HTML::

    build/developer_guide/html/

This output is not bundled into the Windows installer.  It is useful for
browsing the API reference during development.

Build output summary
====================

.. list-table::
   :header-rows: 1
   :widths: 45 55

   * - Path
     - Contents
   * - ``build/manual/html/``
     - User guide, multi-page HTML
   * - ``build/manual/singlehtml/``
     - User guide, single-page HTML (bundled into installer)
   * - ``build/developer_guide/html/``
     - Developer / API guide, multi-page HTML

Sphinx configuration tips
==========================

The ``conf.py`` file at ``docs/manual/source/conf.py`` is standard Sphinx.
A minimal configuration that works well with ``scaldys-builder``:

.. code-block:: python

    project = "myapp"
    copyright = "2024-2026, My Company"
    author = "My Company"

    extensions = [
        "sphinx.ext.autodoc",
        "sphinx.ext.napoleon",
    ]

    html_theme = "sphinx_rtd_theme"

RTD theme
---------

``scaldys-builder`` uses ``sphinx_rtd_theme`` by default in the documentation
built for its own docs.  If your ``conf.py`` already specifies a different
theme, Sphinx will use yours — ``scaldys-builder`` does not override
``conf.py`` settings.

Autodoc
-------

To get meaningful content in the developer guide, enable ``sphinx.ext.autodoc``
in your ``conf.py`` and write NumPy- or Google-style docstrings in your source
modules.  The ``sphinx.ext.napoleon`` extension translates both styles to RST
automatically.
